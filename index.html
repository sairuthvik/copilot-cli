01 — Foundation

What is GitHub Copilot CLI?

Think of it as an AI developer sitting next to you — except it lives in your terminal and can actually make code changes on your behalf.

🤖

AI-powered assistant

Powered by Claude (Sonnet) by default. Understands your codebase, answers questions, writes and edits code.

💻

Terminal-native

Lives in your command line. No browser tab switching, no copy-pasting — it works directly where code lives.

🔗

GitHub-integrated

Knows about your repos, PRs, issues, and branches. You can interact with GitHub workflows via plain English.

👁️

You stay in control

Preview every action before it happens. Nothing is committed or changed without your explicit approval.

"It's not about replacing engineers — it's about removing the communication lag between what PMs need to understand and what engineers need to explain."
02 — Why it matters to you

What's in it for a PM?

You don't need to code. But the closer you are to the codebase, the faster you ship. Here's how Copilot CLI bridges that gap.

🔍

Understand the codebase

Ask "How does authentication work?" and get a plain-English answer — without bothering your engineers.

📋

Review PRs with context

Use /review to get a code review summary before your team meeting. Know what changed and why it matters.

🚀

Create repos & automate setup

Spin up new projects via the GitHub API. Scaffold boilerplate. Reduce time from idea to first commit.

📊

Research before planning

Use /research to deep-dive into GitHub search and web sources — great for competitive analysis or technical scoping.

📝

Document & share

Use /share to export findings to a Gist or Markdown — share technical insights with non-technical stakeholders.

🗺️

Plan before building

Use Plan mode (Shift+Tab) to draft an implementation approach. Review it with engineers before any code is written.

03 — Step-by-step

Getting Started

Follow these steps to go from zero to productive in under 10 minutes.

1

Install GitHub Copilot CLI

Run the install script in your terminal. You'll need an active GitHub Copilot subscription.

Terminal — Install
# macOS / Linux
$ curl -fsSL https://gh.io/copilot-install | bash

# Or with Homebrew
$ brew install copilot-cli

# Windows (WinGet)
$ winget install GitHub.Copilot
2

Log in to GitHub

Launch Copilot CLI and authenticate with your GitHub account using the /login command.

Terminal — Login
$ copilot
🤖 Welcome to GitHub Copilot CLI!

> /login
Opening browser for authentication...
✓ Logged in as @yourusername
3

Navigate to your project

Open your terminal, cd into a project folder, then launch Copilot CLI. It will understand your codebase automatically.

Terminal — Navigate
$ cd ~/projects/my-app
$ copilot
Working in: ~/projects/my-app
✓ Ready. Ask me anything about your code!
4

Ask your first question

Just type in plain English. No special syntax needed. Try something relevant to your current sprint or product area.

Terminal — First Query
> How does user authentication work in this codebase?

🤖 This app uses JWT-based auth. Here's the flow:
1. User submits credentials to /api/auth/login
2. Server validates and returns a signed JWT token
3. Token is stored in HttpOnly cookie for security
4. Each request validates token via middleware...

💡
PM Tip: You can ask follow-up questions in the same session — Copilot CLI remembers the conversation context, just like chatting with a teammate.
5

Try Plan Mode

Press Shift+Tab to switch to Plan Mode — great for scoping work before handing off to engineers.

Terminal — Plan Mode
[ Plan Mode enabled — Shift+Tab to exit ]

> Add a feature to let users export their data as CSV

📋 Implementation Plan:
1. Add /api/export endpoint
2. Query user records from DB
3. Format as CSV using existing utils
4. Return with Content-Disposition header
Estimated complexity: Medium (2–3 days)
04 — Quick Reference

Commands for PMs

The most useful slash commands and keyboard shortcuts, curated for product work.

⌨️  Slash Commands
Command What it does Best for
/plan Create an implementation plan before any code is written PM
/review Analyze recent code changes and surface what matters PM
/research Deep research using GitHub search + web sources PM
/diff Review changes in the current directory All
/pr Operate on pull requests for the current branch All
/share Export the session to a Gist or Markdown file PM
/model Switch AI model (Claude Sonnet, GPT-5, etc.) All
/delegate Send task to GitHub Copilot Agent to create a PR Dev
/context See how much of the conversation context is being used All
/compact Summarize conversation to free up context window All
/help Full list of all available commands All

⌨️  Keyboard Shortcuts
Shortcut What it does
Shift+Tab Toggle between Interactive and Plan mode
↑ / ↓ Navigate command history
Ctrl+C Cancel current operation or clear input
Ctrl+L Clear the screen
Ctrl+D Shut down the CLI
Esc Cancel the current operation

06 — Pro Tips

Tips to get the most out of it

Quick wins for your first week using Copilot CLI as a PM.

🗣️

Talk like a human

No jargon needed. "Why is the dashboard slow?" works just as well as any technical query.

📎

Mention files with @

Type @filename to include a specific file in your context. Great for asking about a specific config or component.

🔄

Keep conversations going

Follow-up questions work. "Now explain just the payment part" after a broad answer drills into detail.

⏪

Undo with /rewind

If Copilot makes a file change you didn't want, /rewind reverts it instantly. No risk of losing work.

🧠

Use /compact regularly

For long sessions, use /compact to summarize history and free up the context window.

🔒

You're always in control

Copilot will ask before doing anything destructive. You approve every action — nothing happens behind the scenes.
