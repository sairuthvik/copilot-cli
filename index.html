Step 1 · What is it

GitHub Copilot CLI — in plain English

You don’t need to be an engineer to understand this. Here’s the one-sentence version:

💡
Copilot CLI is an AI assistant that lives in your terminal. You type a question in plain English and it writes the technical command for you — or explains one that your engineers sent you.

As a PM you’ll use it for two things:

🔍
Understand what’s going on

Ask it to explain commands, error messages, or code your engineers shared. No more 20-minute Google sessions.

⚡
Get things done faster

Generate commands to check logs, search repos, or run scripts without waiting for a teammate.

ℹ️
What it is NOT: it’s not a chatbot in a browser tab. It runs in your computer’s terminal (the black/white window with a blinking cursor). The next section walks you through opening one and getting set up.
Step 2 · Install

Installing Copilot CLI

Three things to install, in order. Each step tells you how to check if you already have it — so you won’t repeat work you’ve already done.

A
Open your terminal
The text interface where you’ll type commands

Pick your operating system:

🍎
macOS
Press ⌘ Space, type Terminal, Enter
🪟
Windows
Press Win + X, choose Terminal
🐧
Linux
Press Ctrl + Alt + T
✅
You’ll see a line ending in $ or % or >. That’s the prompt — it means it’s waiting for input. You’re in the right place.
B
Install GitHub CLI (gh)
Copilot CLI runs as a plugin inside this — install it first

First, check if you already have it:

terminal
$ gh --version
✓ See a version number?
You already have it — skip to step C.
✗ Says “command not found”?
Follow the install options below.

Install GitHub CLI — choose your method:

terminal
$ brew install gh
Don’t have Homebrew installed?
  • Homebrew is a free package manager for macOS. Search “how to install Homebrew mac” on Google
  • Official site is brew.sh — copy the one-line install command from there
  • Or use the Direct download tab instead — no Homebrew needed
powershell
# Option 1: winget (built into Windows 10/11)
$ winget install --id GitHub.cli

# Option 2: Chocolatey
$ choco install gh
Not sure which option to use?
  • Try winget first — it’s pre-installed on most Windows 10/11 machines
  • If you see “winget is not recognized”, search “how to install GitHub CLI Windows”
  • Or use the Direct download tab — works on all platforms
terminal
# Ubuntu / Debian
$ sudo apt install gh

# Fedora / RHEL
$ sudo dnf install gh

# Arch
$ sudo pacman -S github-cli
Distro not listed or getting errors?
  • Search “install GitHub CLI linux [your distro]” on Google
  • Official install page: cli.github.com
📥
Go to cli.github.com and download the installer for your OS. Works on all platforms with no package manager needed.
✅
After installing, run gh --version again. You should see gh version 2.x.x. That means it worked.
C
Install the Copilot extension
One command — plugs Copilot into the GitHub CLI you just installed
terminal
$ gh extension install github/gh-copilot
✅
Verify with gh copilot --version. If you see a version number, you’re good to go.
⚠️
If it says “gh: command not found”, go back to step B — the GitHub CLI wasn’t installed successfully yet.
D
Log in to GitHub
Connects the CLI to your GitHub account and activates Copilot
terminal
$ gh auth login

It’ll ask a few questions. Choose these answers:

? Where do you use GitHub?  ✓ GitHub.com
? What is your preferred protocol?  ✓ HTTPS
? How would you like to authenticate?  ✓ Login with a web browser

A browser window opens. Log in with your GitHub account. Come back to the terminal when done.

✅
Run gh auth status — you should see “Logged in to github.com” with your username.
🎉
Done with install? You only ever need to do this once. From now on, Copilot CLI is available every time you open a terminal.
Step 3 · First run

Your first 5 minutes with Copilot CLI

Open a fresh terminal and try these in order. This is your “hello world” moment.

1
Ask your first question
Use gh copilot suggest to get a command from plain English
terminal
$ gh copilot suggest "list all files in the current folder"

Copilot responds with the command, then asks if you want to run it, copy it, or explain it first.

💡
You can type anything in plain English. Try “show me recent git commits” or “find all .log files”. There’s no wrong phrasing.
2
Explain a command you’ve seen
Paste any scary-looking command and get a plain English breakdown
terminal
$ gh copilot explain "git rebase -i HEAD~3"

Great for reviewing pull requests or understanding what a script does before you share it with stakeholders.

3
Set up a shortcut (optional but recommended)
Type ?? instead of gh copilot suggest every time
terminal
# Run once to activate the shortcut
$ eval "$(gh copilot alias -- bash)"  # swap bash for zsh if needed

# Now use ?? instead of gh copilot suggest
$ ?? "show disk usage"
⚠️
This shortcut resets when you close the terminal. To make it permanent, add the eval line to your ~/.zshrc (or ~/.bashrc) file. Not sure how? Search “how to add alias to zshrc”.
powershell
$ gh copilot alias -- powershell | Invoke-Expression

# Then:
$ ?? "show disk usage"
Step 4 · Commands

The two commands you’ll use 90% of the time

Copilot CLI has two core commands. Here’s what they do and when to reach for each one.

Command 1

gh copilot suggest

Describe what you want to do. Get back a command you can run or copy.

terminal
$ gh copilot suggest "..."
Try asking
→ “find all TODO comments in a project”
→ “show which process is using port 3000”
→ “compress a folder into a zip file”
Command 2

gh copilot explain

Paste any command or error message. Get a plain English explanation.

terminal
$ gh copilot explain "..."
Try pasting
→ curl -X POST -H 'Content-Type: application/json'
→ EACCES: permission denied, open '/etc/hosts'
→ kubectl rollout restart deployment/api

The interactive menu — what your choices mean

Run this command Executes it immediately in your terminal
Copy to clipboard Copies it so you can paste it elsewhere (Slack, a doc, etc.)
Explain this command Breaks it down before you commit to running it — use this when unsure
Revise the command Add more context to refine what Copilot suggests
Step 5 · For PMs

Real PM scenarios — what to actually ask

These are the situations PMs run into every week. Copy and adapt the prompts for your team’s context.

🔍 Understanding a pull request

Your engineer says “check the diff in the feature branch.” You want to know what actually changed — in human terms.

terminal
$ gh copilot suggest "show git changes in a branch vs main"
terminal
$ gh copilot explain "git diff main...feature/checkout --stat"

🐛 Something broke in production

You need to check logs or find when something changed — fast, without waiting for an engineer.

terminal
$ gh copilot suggest "search for error messages in a log file"
terminal
$ gh copilot suggest "show git commits from the last 24 hours"

📊 Velocity check before sprint planning

You want a quick picture of what shipped recently without opening Jira or chasing teammates.

terminal
$ gh copilot suggest "list merged pull requests in the last 2 weeks"
terminal
$ gh copilot suggest "count git commits per author this month"

🤝 Bridging the communication gap

An engineer dropped a command in Slack. You want to understand it before the next call so you can ask a useful question.

terminal
$ gh copilot explain "docker compose up --build -d"
terminal
$ gh copilot explain "kubectl rollout restart deployment/api"
